User Management

Create user

Only the admin can create a new user and delete/disable an existing user. While creating a new user, the
admin can assign different roles and groups to that user.

Following are the steps to create a new user :

  • Login as Admin.
  • Go to Admin->Administration->Manage Users.
  • Click on 'Create New' to create a new user.
  • Enter the details of the user under five categories listed below:
    • Ø Personal Details
    • Ø Job Details
    • Ø Contact Details
    • Ø Permissions
    • Ø Others
  • ñ Click on 'Save' to create the user.

Edit User

The details of a user can be edited by both and admin and the user concerned. The user roles, group
and department can be edited by the admin only. Any user can be deleted by the admin.

Following are the steps to edit user details :

For Admin

  • Login as Admin.
  • Go to Admin->Administration->Manage Users.
  • Click on any user name on the list to edit details for that user.
  • Click on the check box to select user(s).
  • Click on 'Delete' to remove the selected user(s).
  • Click on 'Disable' to disable the selected user(s).

For User

  • Login as User.
  • Click on the name of the displayed on the top right corner to edit the details.

Role Management

Create Role

The admin can create a new role. While creating a new role, the admin specifies the type i.e. public role or department role. For the department role, admin has to select the department to which that role belongs.

Following are the steps to create a new role :

  • Login as Admin.
  • Go to Admin->Administration->Manage Roles.
  • Click on 'Create New' to create a new role.
  • Enter the name of the role.
  • Select the type i.e. Public Role or Department Role.
  • In case of 'Department Role', select the department for that role.
  • Give a description of the role.
  • Click on 'Save' to create the role

Update Role

Following are the steps to update an existing role :

  • Login as Admin.
  • Go to Admin->Administration->Manage Roles.
  • Click on the name of any role.
  • Change the details for the role.
  • Click on 'Update' to save the changes.

Delete Role

Admin can delete any existing role. However, roles which belong to type 'System' cannot be deleted.

Following are the steps to delete an existing role :

  • Login as Admin.
  • Go to Admin->Administration->Manage Roles.
  • Click on check-box to select any role(s).
  • Click on 'Delete' to remove the selected role(s).

Group Management

Create Group

The admin can create a new group. A group can either be a parent group or it can be sub-grouped under some other super-group.

Following are the steps to create a new group :

  • Login as Admin.
  • Go to Admin->Administration->Manage Groups.
  • Click on 'Create New' to create a new group.
  • Enter the name of the group.
  • Click on 'IsParent' check-box if the group is going to be a parent group.
  • If not a parent group, then select a super group under which the new group will belong.
  • Select the person in-charge, leader, administrators and interface people for the group.
  • Enter description and remark and then click on 'Save' to create the group.

Update Group

Following are the steps to update an existing group :

  • Login as Admin.
  • Go to Admin->Administration->Manage Groups.
  • Click on the name of any group.
  • Change the details and click on 'Update'.

Delete Group

Following are the steps to delete an existing group :

  • Login as Admin.
  • Go to Admin->Administration->Manage Groups.

Department Management

Create Department

The admin can create a new department. A department can either be a parent department or it can be placed under some other department. If a department is deleted, the admin has the option of selecting whether the users of that department will also be deleted or they will be moved to the parent department.

Following are the steps to create a new department :

  • Login as Admin.
  • Go to Admin->Administration->Manage Departments.
  • Click on 'Create New' to create a new department.
  • Enter the name of the department.
  • Click on 'IsParent' check-box if the department is going to be a parent department.
  • If not a parent department, then select a super department under which the new department will belong.
  • Select the owner, administrators and interface people for the department.
  • Select whether the users will be deleted or moved to parent department when the department is deleted.
  • Enter a description of the department.
  • Click on 'Save' to create the department.

Following are the steps to update an existing role :

  • Login as Admin.
  • Go to Admin->Administration->Manage Departments.
  • Click on the name of any department.
  • Change the details for the department and click on 'Update'.

Delete Department

Following are the steps to delete an existing role :

  • Login as Admin.
  • Go to Admin->Administration->Manage Departments.
  • Click on check-box to select any department(s).
  • Click on 'Delete' to remove the selected department(s).

Menu Management

Create Menu

The admin can create a new menu item in the top menu or side menu. The menu item can be a Parent menu item or it can be a sub menu item for an already existing parent menu. The top menu item can only be the parent menu. The side menu item can be either parent menu item or side menu item.

Following are the steps to create a new menu :

  • Login as Admin.
  • Go to Admin->Administration->Manage Menus.
  • Click on 'Create New' to create a new menu item.
  • Enter the name of the menu item.
  • Enter the help text for the menu item.
  • Select the URL type (URL or Script).
  • Enter the URL to which the user will be redirected after clicking on menu item.
  • Enter the order to specify the position of the menu item.
  • Click on 'IsParent' check-box if the menu item is going to be a parent item.
  • Select whether the menu item is going to appear on side menu or top menu. If the 'IsParent' option is not selected, then it can only be located in side menu.
  • If 'IsParent' and 'Side Menu' options are selected, then select 'Parent Menu' to specify under which top menu item will the new menu item appear.
  • If the 'Side Menu' option is selected without 'IsParent', then select 'Parent Menu' to specify the parent menu item on the side menu.
  • There is no 'Parent Menu' option for 'Top Menu' item.
  • Select the roles for which the new menu item will be displayed.
  • Choose the icons for the menu item when it is selected and unselected.
  • Click on 'Active' check-box to activate the menu item.
  • Enter description then click on 'Save' to create the menu item.
  • After the menu item is created, the admin is logged out automatically.

Update Menu

Following are the steps to update an existing menu item :

  • Login as Admin.
  • Go to Admin->Administration->Manage Menus.
  • Select any menu item from 'Header' or 'Top' menus.
  • Change the details and click on 'Save.
  • After the menu item is updated, the admin is logged out automatically.

Create a new table

Create a new table

Only admin can create a new table

Following are the steps to create a new table:

  • Login as Admin.
  • Go to Process->Tables->Create new.
  • Fill all the Mandatory fields
  • Drag and Drop the Parent table or Child table and save.

The Fields provided in the Tables are listed as below :

  • Table Name
  • Chinese name for table
  • Column name
  • Chinese name for Column
  • Type of the Column.
  • Length of Column - Allows only given length while starting the Process.
  • Auto Generation ID - Create ID in DB depends upon the given Condition.

Edit Table

The details of a table can be edited by admin. Any table can be deleted by the admin.

Following are the steps to edit table details :

  • Login as Admin.
  • Go to Process-> Form-> Tables-> Edit Table
  • Click on any table name on the list to edit details for that table.
  • To delete the table select any table and Click on delete

Export CSV

Export CSV helps to export the whole information from the table in Excel format.

Admin can see/refer the data availabe in Exported Table

Following are the steps to Export CSV :

  • Login as Admin.
  • Go to Process-> Form-> Tables-> Export CSV
  • Click on any table name on the list to Export CSV

Export SQL

Export SQL helps to export the whole information from the table as SQL Dump for Backup Purpose .

Following are the steps to Export SQL :

  • Login as Admin.
  • Go to Process-> Form-> Tables-> Export SQL
  • Click on any table name on the list to Export SQL

Import SQL

Importing the Exported SQL Dump without selecting any table from local drive inserts the old data in Particular table which is used when data lost.

Following are the steps to Import SQL :

  • Login as Admin.
  • Go to Process-> Form-> Tables-> Import SQL
  • Click on Import SQL and Import the SQL Dump from Local Drive.
  • This will automatically insert the data in particular Table.

Import CSV

Importing the Exported CSV from local drive inserts the old data in Particular table which is used when data lost.

Following are the steps to Import CSV :

  • Login as Admin.
  • Go to Process-> Form-> Tables-> Import SQL
  • Click on Import CSV, Select the Table need to Import and Import the CSV from Local Drive.
  • This will automatically insert the data in particular Table.

Process Operations

Only admin can view and the list of processes. The created workflow is listed under 'Process List'.

The updated process is stored as new active version. Previous versions can be restored/deleted in the

'Show Versions' page. The admin can also export or import a process as XML file.

Edit a Process

  • Login as Admin.
  • Go to Process->Manage-> Process List to view the list of processes.
  • Click on any process name to edit that process.
  • The updated process becomes the latest version.

Delete

  • Login as Admin.
  • Go to Process->Manage->Process List.
  • Click on check box to select any process(es).
  • Click on 'Delete' to delete the selected process(es).
  • On deleting a process, all the versions for that process will be deleted.
  • Click on 'Show Versions' to view all versions of that process.
  • Click on 'Delete' to delete any version.

Export

  • Login as Admin.
  • Go to Process->Manage->Process List.
  • Click on 'View/Export Preview' to preview the XML code for any process.
  • Click on check box to select any process(es).
  • Click on 'Export' to export the selected process(es) as XML file.
  • Click on 'Show Versions' to view all versions of that process.
  • Click on 'View/Export Preview' to preview the XML code for any version.
  • Click on check box to select version(s).
  • Click on 'Export' to export the selected version(s) as XML file.

Import Process

  • Login as Admin.
  • Go to Process->Manage->Process List.
  • Click on 'Import' to import any XML file from the system.

View Versions

  • Login as Admin.
  • Go to Process->Manage-> Process List.
  • Click on 'Show Versions' to view all versions of a process.
  • The latest version is the active version.
  • The old version can be restored if required by clicking on 'Restore'.

Monitor Workflow

The admin can monitor a process to know how many instances are running and their status. The

active instances can be suspended and the suspended instances can be made active. This can be done only

by the admin.

Users can also monitor their process in "My Monitor process" :

Following are the steps to monitor the workflow :

  • Login as Admin.
  • Go to Process->Manage->Monitor Process to view the list of instances and their status.
  • Click on the count of instances to suspend an active instance or vice versa.
  • Click on 'Total Instances' count to view the status of all the instances for that process.

Start Process Workflow

The user can create an instance of a process by starting the workflow. After starting a process, the user has

to input the details in a form. When the form is submitted, it will follow the sequential flow of the process

(For eg. It will be assigned to a group).

Following are the steps to start a process instance :

  • Login as User or Admin.
  • Go to Process->Process->Process Definition.
  • Click on 'Start' to execute the workflow.
  • After clicking on 'Start' the corresponding form is displayed. Enter the input in the form to create an instance.

Delete Process

User can view/delete/terminate all his instances in 'My Instances' section. He can view the task instances by clicking on any instance name. Deleting the process will delete instances on screen. But its execution records, the user who deleted the process, and reason for deletion will exist.

Following are the steps to delete a process instance :

  • Login as User or Admin.
  • Go to Process->Process->My Instances to view the instances running for the logged in user.
  • Click on 'Delete' to delete the process.

Terminate Process

Terminating the process will completely remove the process from the system along with its execution records.

Following are the steps to terminate a process :

  • Login as User or Admin.
  • Go to Process->Process->My Instances to view the instances running for the logged in user.
  • Click on 'Terminate' to terminate the process.

Export to CSV

The user can export the list of process instances as a CSV file.

Following are the steps to export the process instances to CSV :

  • Login as User or Admin.
  • Go to Process->Process->My Instances.
  • Click on 'Export CSV' to export the instances as CSV.

Claim a Task

When a task is created, it is assigned to a group/user. The task appears in the 'Waiting List' of all the users in

that group. Any user from that group can claim a task. After a task it claimed, it is moved from the

'Waiting List' to the 'Bucket' of the user who claimed it.

Following are the steps to claim a task :

  • Login as User or Admin.
  • Go to Process->Task->Waiting List to view the list of pending tasks that have not been assigned to any user.
  • When the task is created, it is assigned to any group or department but not to any user.
  • Only the users who are eligible to become an assignee of a task can view that task in their 'Waiting List'.
  • Click on 'Claim' to claim a task that user wants to operate.

User Bucket

The user can view all the tasks claimed by him in 'My Bucket'. The user can then either approve/reject

the task or reassign it to some other user. He can also involve more members in the task.

Following are the steps to operate a task :

  • Login as User or Admin.
  • Go to Process->Task->My Bucket to view the tasks claimed by the user.
  • Click on 'Operate' button to approve/reject the task.
  • Alternatively, click on the name of the task to get the detailed view and then click on 'Operate' button.
  • In detailed view, click on 'Reassign' to assign that task to another user.
  • In detailed view, click on 'Involve Members' to involve more users in different roles.

User Involvement

If a user is involved in a task at any point of time, it will be listed in Process->Task->My Involvement'.

Admin can also assign "Assignee" and "Owner" for the task

The roles in which the user can be involved in a task are listed below :

  • Coordinator
  • Creator
  • Organizer
  • Processed User
  • Reader
  • Workflow Administrator
